# Moguria Design Notes

## 変えてはいけない核

- 毎回壊れビルドを探すかわいい地獄。
- 食べて成長する。
- ダンジョンは熱く、ホームは静か。
- プレイヤーに「もう一回」を感じさせる。
- ホームは複雑にしすぎない。

## 現在の開発段階

v1.3では、遊びの追加よりも拡張時に重くならないための基盤を追加した。

## 軽量化方針

- 初回表示に必要なものだけcriticalにする。
- ステージ別素材はpackとして遅延ロードする。
- 音声はユーザー操作後に再生・ロードする前提にする。
- 動画は原則としてゲーム中の必須要素にしない。
- 画像・音・動画を追加する時はPERFORMANCE_BUDGETを確認する。

## 今後の実装方針

- 新スキルや新アーティファクトはデータ追加を優先する。
- 演出追加時は低負荷モードで省略できるようにする。
- ステージ追加時はasset pack単位で読み込む。
- 通信が必要になるまでは完全静的で進める。


## v1.4 仕様固定メモ
- ホームは「呼吸する場所」。作業量を増やさない。
- スキルは数値上昇より、攻撃スタイル/挙動変化を優先。
- アーティファクトは1プレイのビルド軸・救済・固定山場。
- ブラウザ単体では完全なチート防止はできないため、競争要素を入れる時はサーバー検証へ移行する。
- 開発中は `#dev` を使い、公開時は外す。

## v1.5 方針
- スキルは単なる数値強化ではなく、攻撃スタイル・距離感・逃げ方・防御方法が変わるものを優先する。
- アーティファクトは「今回の軸決め」「序盤不運の救済」「壊れビルドの加速」の役割を持たせる。
- 敵はビルド価値を変える存在として設計する。硬い敵、逃げる敵、遠距離敵、分裂敵などで、単一ビルドだけが常に正解にならないようにする。


## v1.8 Meta Progression Foundation
- MoguCoin（通貨）基盤を追加。
- 5部位装備、インベントリ、簡易ガチャ、装備強化の基盤を追加。
- おでかけ/チャレンジ入口を追加。
- 本格育成・サブコンテンツ実装前の軽量な導線として追加。


## v1.9.0 方針

- 敵の密集は「脅威」ではなく「視認性低下」になりやすいため、重なりは軽く押し出して解消する。
- 特殊モブイベントはビルドが未完成の序盤では理不尽にならないよう、硬い敵イベントは中盤以降に限定する。
- 制限時間は緊張感・達成感・集中力を作るためのもの。同時に、極端な逃げ戦法やバグ悪用による想定外クリアを抑える。

## v2.0 演出方針
- 世界観入口はかわいい。プレイ感は爽快。
- 通常敵はかわいく、ボスは少し不穏にする。
- 演出は「優しい光」と「深い影」で構成する。
- テキスト説明より、光・音・動きで伝える。
- 派手にしすぎず、軽やかな爽快感を優先する。

## v2.4 演出方針

演出は「気持ち良さ」を上げるために追加するが、入力遅延や発熱を起こすほど重くしてはいけない。
Moguriaの演出は、派手さよりも、短く、軽く、余韻が残ることを優先する。

特に守ること:
- 危険演出は緊張を作るが、視認性を奪わない。
- 覚醒演出は爽快にするが、操作の邪魔をしない。
- ボス演出は不穏にするが、怖くしすぎない。
- ホームはあくまで静かで安心できる場所にする。

### v2.5 演出方針
- Moguriaの演出は「優しい光」と「深い影」を軸にする。
- 派手さは上げるが、長時間プレイで疲れるギラギラ感は避ける。
- 音は短く軽く、同時発音を抑えて端末負荷と耳疲れを防ぐ。
- ボスは怖さではなく、少し理解できない異質さを出す。
- ホームは演出を足しても複雑にせず、帰還後の安心感を保つ。


## v2.6 Design Notes
- スキルは“取得する”より“食べて育つ”体験へ寄せる。
- 同じスキルの再取得は単なる重複ではなく、ビルドの意思決定として扱う。
- 合体スキルは、狙って完成させた時の快感を作るための中盤以降の目標。
- 封印は運の悪さへの救済であり、ビルド方針を絞るための軽い戦略要素。
- ビジュアルはキービジュアル準拠。“優しい光＋深い影”を維持する。


## v2.7 Visual Direction Lock
- 全体デザインはキービジュアル準拠。暖色の安心感、ラベンダー〜紫の異質感、星の光を共通モチーフにする。
- ノーマル敵はかわいく柔らかい。ボスは花/星/影が混ざった不気味さを少し持つ。
- UIは絵本的な柔らかさと、光るガラス感。文字ではなく光・形・動きで理解させる。


## v2.8 UI Redesign Direction
- UIは単なる操作パネルではなく、Moguria世界の中に存在する星灯りの道具として扱う。
- ホームは『暖かい帰る場所』、バトルは『深い影と星の光』を基調にする。
- 下部ドックはゲームアプリ的な分かりやすさを保ちつつ、宝箱・扉・水晶のようなモチーフで世界観化する。
- テキスト量を増やさず、光・色・アイコン・奥行きで理解させる。
- 既存の機能構造は維持し、今後の本格アセット差し替えに備えてCSS中心で刷新した。


## v2.9 Visual Direction
キービジュアルを最上位の正として、既存UIの延長ではなく「Moguriaの世界の中にUIが浮かぶ」方向へ刷新。
- Mogu: 白く丸く小さい、守りたくなる相棒。
- 通常敵: かわいいが深層では少し異質。
- ボス: 紫の光と歪みで、怖すぎない不気味さ。
- UI: 星灯り、ガラス、宝石、柔らかい金縁。
- エフェクト: 光の粒子、星のかけら、深い紫の影。


## v3.0 Smartphone UI Rebuild
- ホームを「Moguの部屋・家」として再設計。キービジュアル直置きではなく、暖かい室内と深い夜の窓をCSS/Canvasトーンで再構築。
- バトルHUDをスマホ視線優先に整理し、HPはMogu頭上のバーで直感的に確認できる方針へ変更。
- スキルカード・アイコン・ボトム導線をスマホで押しやすいサイズへ再調整。
- Mogu、敵、ボス、レア敵、ドロップ、弾の描画を白いふわふわMogu／星灯り／紫の深層感へ寄せた。

## v3.1 Home Screen Design Lock
今回の変更範囲はホーム画面のみ。今後の大規模刷新は小さく分割して進める。

ホームは「キービジュアルを背景に貼る」のではなく、Moguが帰る暖かい部屋として作る。
基本要素は、夜の窓、星の吊り飾り、ランプ、ベッド、机、小物、ラグ。
UIはゲーム外のメニューではなく、部屋の光に溶け込む星灯りガラスUIとして扱う。

今回触っていないもの：バトルHUD、Mogu戦闘描画、敵、ボス、スキル効果、ゲームロジック。


## v3.1a Art Implementation Note
This version starts the true art-material pipeline. Unlike v3.1, it includes actual PNG assets in `assets/images/home/` and applies them to the home screen. Future art updates should follow this flow: generate asset → store in assets/images/[screen] → register in manifest → apply through CSS/Canvas → verify on smartphone.
