# Moguria Performance Budget

Moguriaはスマホのブラウザで遊ぶ前提のため、機能追加よりも「軽く起動して、遊んでいる間に重くならない」ことを優先する。

## 初回表示
- 初回に必須ロードする素材は2MB以内を目標にする。
- ホーム、Mogu、基本UI、最低限のSEだけをcritical扱いにする。
- ステージ別画像、追加BGM、動画、豪華演出はlazy / pack扱いにする。

## 画像
- 可能な限りCSS/Canvas描画、SVG、スプライトシートを優先する。
- 同じ敵の色違いは別画像を量産せず、Canvas側の色変換や小差分で表現する。
- 大きな背景画像はステージ突入直前に読み込む。

## 音声
- BGMは初回に全読み込みしない。metadataまたはユーザー操作後にロードする。
- SEは短いものをまとめ、将来的にはAudio Sprite化する。
- 同時再生数に上限を設ける。

## 動画
- 原則としてゲーム中の動画再生は避ける。
- 使う場合はホームや図鑑など、操作負荷が低い場所に限定する。
- 自動再生・大容量動画・複数動画同時保持は禁止。

## 通信 / パケット
- 現時点では完全静的・ローカル保存を基本とする。
- 将来アップデートデータを外部取得する場合は、manifestで差分管理する。
- 取得失敗時でも既存バージョンで遊べる設計にする。
- リアルタイム通信を前提にしない。

## 実行時負荷
- 画面外の敵、弾、経験値は更新頻度を落とす。
- パーティクル、爆発、分裂弾には上限数を設ける。
- FPSが落ちた場合はMoguriaPerformanceがqualityを下げる。

## 追加時のルール
新しい演出・画像・音・動画を追加する時は、以下を必ず確認する。

1. 初回ロードに入れる必要があるか
2. lazy loadで良いか
3. 使い終わったら破棄できるか
4. 低スペック時に省略できるか
5. ファイルサイズが妥当か
